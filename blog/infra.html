Infra: Offline rendering for realtime shaders

[2024/12/XX]

Last week I made the source code for Infra, my custom GLSL shader renderer and exporter, open source and freely available. I originally built the program to address a very specific use case that I needed at the moment: Rendering the output of GLSL shaders to arbitrary resolutions and framerates. In this blog post I want to explain more in depth what the program does, why I built it and what you can do with it, along with a short explanation of its usage. Therefore the contents of this post are separated into the following segments:

  • An Offline GLSL Renderer
  • The Use Case
  • Using Infra
  • Extra: Implementation

An Offline GLSL Renderer

Infra is a tool with which you can write your own fragment shaders, load them into the program and export either an image or a video sequence of the output of the shader with an arbitrary resolution and framerate. Admittedly, this is not a good way to summarize what the program does or what it is for for those who are not familiarized with computer graphics terminology, [so I'm now going to give a small introduction to the concepts in order to understand]. [Talk about what you can do with shaders: Digital art, music videos, VJ loops, motion graphics, 3D textures] [History on why I made it, there are no good solutions out there for this specific use case] In short, shader code goes in, image or video comes out.
